# Skill：SDD 需求概述

## 触发词
- 需求概述
- 功能分析
- 业务背景
- 需求规格

## 适用场景
SDD 文档编写初期、项目立项阶段、需求评审前准备

## 禁忌场景
技术细节设计、代码实现、测试用例编写

## 执行步骤
1. 提取业务目标与核心价值
2. 分析用户角色与使用场景
3. 识别核心功能需求
4. 定义非功能需求（性能、安全、可扩展性）
5. 输出结构化需求概述章节

## 输出模板
```markdown
## 1. 需求概述

### 1.1 项目背景
{项目业务背景描述}

### 1.2 业务目标
- {目标1}
- {目标2}

### 1.3 用户角色
| 角色 | 职责 |
|------|------|
| {角色名} | {职责描述} |

### 1.4 核心功能需求
| 需求编号 | 功能描述 | 优先级 |
|----------|----------|--------|
| REQ-001 | {描述} | P0 |

### 1.5 非功能需求
- 性能：{性能指标}
- 安全：{安全要求}
- 可扩展性：{扩展要求}
```

## 校验标准
- 包含完整的背景、目标、角色、需求四个维度
- 需求描述清晰、可验证
- 优先级标注明确
- 非功能需求量化可衡量

## 异常处理
- 需求模糊时返回「请提供项目业务背景、核心功能、用户角色等关键信息」
- 信息不完整时提示缺失项并请求补充
